# 2026-03-23 - Emergency Cart Crash Fix

## 🚨 CRITICAL BUG FIXED: Wheel add-to-cart crash

### Root Cause
**React Rules of Hooks violation in CartSlideout.tsx**

The `useState` and `useEffect` hooks for `liftedCtx` were called AFTER the early return:
```tsx
if (!isOpen) return null;  // Early return here

// WRONG: Hooks called after conditional return
const [liftedCtx, setLiftedCtx] = useState<LiftedBuildContext | null>(null);
useEffect(() => { ... }, [isOpen]);
```

This violates React's fundamental rule: **Hooks must be called in the same order on every render**.

When cart was closed (`isOpen=false`), component returned early - no hooks called.
When cart opened (`isOpen=true`), React encountered unexpected hook calls and crashed.

### Fix Applied
Moved the `useState`/`useEffect` hooks BEFORE the early return:
```tsx
// CORRECT: Hooks before any conditional returns
const [liftedCtx, setLiftedCtx] = useState<LiftedBuildContext | null>(null);
useEffect(() => {
  if (isOpen) {
    setLiftedCtx(loadLiftedContext());
  }
}, [isOpen]);

if (!isOpen) return null;  // Early return after hooks
```
