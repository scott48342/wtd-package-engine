## Wheel Availability Cache Pre-Warm Implementation

### Task
Implement pre-warm cache system for wheel availability to reduce search response times for common truck/SUV searches.

### Files Created
1. **`src/lib/availabilityCache.ts`** - Centralized availability cache module
   - Cache key format: `${sku}|minQty=${minQty}`
   - 30-minute TTL (configurable via `WT_AVAIL_CACHE_TTL_MS`)
   - 10,000 max entries with LRU eviction
   - Tracks hits, misses, pre-warm hits

2. **`src/lib/availabilityPrewarm.ts`** - Pre-warm job system
   - Targets high-frequency truck/SUV bolt patterns
   - Priority 1: F-150 (6x135), Silverado 1500 (6x139.7), Ram 1500 (5x139.7)
   - Priority 2: Wrangler (5x127), Tacoma (6x139.7), Tahoe (6x139.7), Sierra 1500 (6x139.7)
   - Priority 3: Tundra (5x150), Super Duty (8x170), Grand Cherokee (5x127)
   - Concurrency-controlled API calls (8 concurrent, configurable)
   - 5-minute max job time budget

3. **`src/app/api/warmup/availability/route.ts`** - API endpoint
   - `?action=status` - Get cache stats
   - `?action=run` - Trigger pre-warm job
   - `?action=start-scheduler` - Start 25-minute interval scheduler
   - `?action=stop-scheduler` - Stop scheduler
   - `?action=clear-cache` / `?action=reset-metrics`

4. **`scripts/prewarm-availability.ts`** - CLI script
   - `npx tsx scripts/prewarm-availability.ts [--dry-run] [--targets "F-150,Silverado"] [--max-skus 100]`

5. **`scripts/validate-prewarm.ts`** - Validation/test script
   - Compares cold vs warm performance for test vehicles

6. **`docs/availability-cache-prewarm.md`** - Full documentation

### Files Modified
1. **`src/app/api/warmup/route.ts`** - Added availability pre-warm support
   - `?availability=1` triggers pre-warm
   - `?quick=1` skips WheelPros API calls

2. **`src/app/api/wheels/fitment-search/route.ts`** - Uses centralized cache
   - Replaced local availCache with imported module
   - Added `availabilityPrewarmHits` tracking in timing response

### Cache Strategy
- **Schedule:** Every 25 minutes (5 min before 30-min TTL expiry)
- **Target:** Top 200 SKUs per bolt pattern
- **Concurrency:** 8 parallel API calls
- **Rate limiting:** 100ms delay between batches

### Expected Impact
- Cold start searches: ~2-5 seconds
- Warm cache searches: ~200-500ms for targeted vehicles
- Pre-warm targets cover ~80% of truck/SUV traffic

### Verification
- TypeScript compiles without errors
- Cart/checkout flow unaffected (availability check same interface)
- Fitment validation unchanged (pre-warm only affects availability lookups)
