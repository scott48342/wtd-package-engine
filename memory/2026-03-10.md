# 2026-03-10

## WTD sandbox UI + package engine progress

- Added Google Places API key to `wtd-package-engine/.env` and wired it into Installer lookup.
  - `GOOGLE_PLACES_API_KEY` config added.
  - Installer lookup now falls back to Google Places Nearby Search when DB has no installers.
  - Tuned search to use keywords **"tire shop"** and **"tire store"** within **15 miles**.

- Fixed React dev UI → backend calls by enabling CORS in the package engine (`CORS_ALLOW_ORIGIN`, default `http://localhost:3000`).

- Built out the sandbox React UI (`wtd-sandbox-ui`) into 3 tabs:
  - **Tires**: vehicle → OEM tire size → `/v1/tires/search`
  - **Wheels**: vehicle fitment → `/v1/wheels/search`
  - **Packages**: `/v1/packages/plus-size`

- Fixed WheelPros tire search adapter so `size=215/55R18` is parsed into `width/aspectRatio/wheelDiameter` params.

## Trim/modification-aware fitment

- Added trim support using Wheel-Size "modifications":
  - New endpoint: `GET /v1/vehicles/trims?year=&make=&model=`
  - Updated `GET /v1/vehicles/search` to accept `modification` (+ optional `trim`) and return fitment fields + `vehicleModificationId`.
  - DB/persistence updated to cache fitment per (vehicle, modification): new `vehicle_modification` table and `vehicle_fitment.vehicle_modification_id`.
  - Packages now accept `vehicleModificationId` so plus-size recommendations are trim-correct.

- Improved trims endpoint to de-dupe by trim label and pick the "best" modification (trim_scoring desc, stable tie-breaker).

## UI upgrades (Tire Rack-style direction)

- Wheels/Packages grid upgraded with:
  - diameter chips (All + available diameters)
  - sort dropdown (Featured/Price/Brand)
  - improved product tiles (brand/title, big MSRP, size line like `20x9 +20`).
