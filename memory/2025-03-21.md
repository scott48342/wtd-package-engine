# 2025-03-21 Session Notes

## DB-First Fitment System - Major Progress

### Bug Fix: Decimal Offset
- **Error**: `invalid input syntax for type integer: "44.45"` for 2009 Jeep Wrangler
- **Root cause**: `offset_min_mm`/`offset_max_mm` columns were `integer`, but Wheel-Size API returns decimals
- **Fix**: Changed columns to `decimal(5,2)`, updated all normalizers to convert to strings
- **Migration**: `drizzle/migrations/0002_offset_decimal.sql`

### Bug Fix: Wrong API Endpoint
- **Problem**: `profileService.ts` was calling `vehicles/` endpoint which returns empty `technical` data
- **Fix**: Changed to `search/by_model/` endpoint (same as legacy system)
- **Result**: Now gets full data: bolt_pattern, centre_bore, thread_size, wheel setups

### DB Connection Fix
- Switched from `@vercel/postgres` to `node-postgres` (`drizzle-orm/node-postgres`)
- Better compatibility across environments

## Accessory Fitment Service (NEW)

### Created: `src/lib/fitment/accessories.ts`

**Functions:**
- `getAccessoryFitment(vehicle, wheel)` → full recommendations
- `parseThreadSize(raw)` → handles M14x1.5 and 1/2"-20 formats
- `calculateHubRingSpec(vehicleHub, wheelBore)` → determines if ring needed
- `normalizeSeatType(raw)` → handles conical/ball/flat/mag aliases
- `formatAccessoryForUI(rec)` → badge + headline + explanation

**Logic:**
- **Lug Nuts**: REQUIRED when `threadSize` available from dbProfile
- **Hub Rings**: REQUIRED when `wheel.centerBore > vehicle.centerBoreMm`
- **Skipped**: When data missing (never guess)

**Test Results:**
```
Wrangler:  Lug nuts REQUIRED (1/2"-20), Hub rings REQUIRED (78→71.6mm)
Silverado: Lug nuts REQUIRED (M14x1.5), Hub rings REQUIRED (106.1→78.1mm)
Camaro:    SKIPPED - no vehicle data
OEM Wheel: Lug nuts REQUIRED, Hub rings NOT NEEDED (bore matches)
```
